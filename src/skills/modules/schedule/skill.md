# Schedule

建立一次性提醒或未來任務，將排程寫入 Golem 的排程資料，讓系統可以在指定時間提醒使用者。

## 使用時機

- 使用者要求「提醒我」、「幫我排程」、「明天叫我」、「某時間做某事」。
- 使用者給出明確任務內容與未來時間。
- 使用者想把對話中的某件事轉成提醒。

## Action 格式

```json
{
  "action": "schedule",
  "parameters": {
    "task": "提醒內容",
    "time": "2026-05-04T09:00:00+08:00"
  }
}
```

## 注意事項

- `time` 必須是可解析的未來時間，最好包含時區。
- 若時間模糊，先向使用者確認日期與時間。
- 若只是要查看既有排程，請改用 `list-schedules` 技能。
